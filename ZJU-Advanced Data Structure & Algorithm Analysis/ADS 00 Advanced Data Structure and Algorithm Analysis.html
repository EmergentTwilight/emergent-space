    CS大二下课程回忆&学习总结(数据库系统 计算机组成｜计组 高级数据结构｜ADS ITP) - CC98论坛

    杨洋老师特别 nice，由于笔者有一定的算法基础，因此整体来说不算特别折磨。但是这课有一种温水煮青蛙的感觉，平时的课程作业没有什么代码，因此很多人就会在平时忽略掉这门课程（minisql 导致的，我们小组是把所有的 project 都做了，因此平时分都拿满了，并且 project 还可以在一定程度上监督你复习本周学习的知识，所以还是比较推荐全部做完的，我们小组采用的是环形队列的方式，每次就由一个人负责写代码，另外一个人负责写报告，然后下一次交替，中间如果有人有事情，再选择其他人进入，最后基本不累的做完所有 project 了。主要参考资料是修佬的笔记还有 wyy 的 ads 笔记，wyy 的 ADS 笔记基本覆盖完了所有的内容，十分推荐。 今年期末考试的难度不是特别大，所以可以看出 ads 的难度正在逐年下降的，因此不需要特别担心。平时认真听杨洋老师的课堂，或者看 myc 的智云，期末考试之前刷几套历年卷基本就问题不大了

    CSer 大二下课程回忆&资源（数据库系统DB，ADS，计组，OOP，数据建模与分析） - CC98论坛

    期末

    ADS 的期末应该是计院学子的梦魇之一了，特别是还有个 yds 出题。好在这两年叶老师逐渐收手，题目开始更加贴近课内所学，难度有所下降。今年的题目也是如此，叶老师考前宣称：”今年我都没出几道题，你们考得不好不能怪我了。“
    今年来说，体感题目整体比去年难，简单题没有这么多，但是编程题把分送到位了（去年 AVL[ac01]），而且没有很难的题，中档题的比例变多了。

    关于这门课的期末复习，笔者认为可以从这几点去准备

    • 平时听课应该认准张叶毛三人的课，他们是做算法的，本身会补充 PPT 没有的东西。期末最难的题大概率也来自他们之手。比如考前几天很多同学在问 Load Balancing 的近似比，我记得讲近似算法这节课的最后，叶老师提到了这个问题并讲了大致的思路。
    • 算法部分，可以看看《算法设计》这本书，上面对于 P&NP、近似算法、局部搜索、随机算法的讲解都非常到位。
    • 要把 PPT 过一遍。
      • 数据结构部分的代码要非常熟悉，了解变量的含义。（姥姥的题有可能不会给有效的注释，得凭自己猜）
      • PPT 的重点不是很突出，因此需要细致地看每句话，不要漏掉细节。
      • 部分章节编的不是很好（最差的就是 P&NP 了，上来讲的不知所云…个人认为从多项式归约开始讲会好得多）
    • 多做题，这几年的历年卷/练习题资源应该足够丰富了。不会的及时问老师或者助教，因为我感觉不问他们其他人几乎都做不出来，还是得问问专业的人（雾）。

    放平心态，不要太紧张。笔者考前模拟题没怎么上过 60，考试的时候感觉题比去年难好多要完蛋了，结果最后考完发现 90 分 [ac01]，一瞬间不敢相信这是自己的期末成绩。祝后面的同学好运，愿天堂没有 yds 和他出的题。
